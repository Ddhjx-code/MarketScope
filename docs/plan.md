# MarketScope 项目详细规划

## 项目概述

基于每日股票分析系统（daily_stock_analysis）项目架构，开发一个行业分析AI应用。该应用能够根据用户输入的APP类型和功能描述，分析相关行业现状，生成详细的市场分析报告。

## 核心功能

### 1. 输入处理
- 用户输入：APP类型 + 功能描述
- 示例：`"社交类APP，主要功能是兴趣小组聊天"`

### 2. 数据增强（RAG）
- **市场规模分析**：获取行业规模、用户数量、增长趋势
- **竞品分析**：识别主要竞争对手、功能对比、市场份额
- **趋势分析**：分析行业发展趋势、技术方向、政策影响
- **用户需求分析**：分析用户痛点、需求变化、行为模式

### 3. 综合分析
- 市场容量评估
- 竞争格局分析
- 进入机会识别
- 风险因素评估

### 4. 报告生成
- 结构化的行业分析报告
- 市场进入可行性评估
- 建议策略和风险提示

## 数据获取策略

### 1. 搜索引擎 API（RAG 核心）
- **Tavily API**：用于获取最新的市场趋势、行业分析报告
- **Serper API / SerpAPI**：获取竞品信息、市场数据
- **Google Custom Search**：补充搜索功能

### 2. 专业金融/行业数据 API
- **Alpha Vantage**：行业数据和报告
- **FRED (Federal Reserve Economic Data)**：宏观经济指标
- **IBES/EPS Data**：行业盈利预期
- **S&P Capital IQ** 或其他商业数据库：行业基准数据（如果有访问权限）

### 3. 公开数据源 API
- **政府统计部门 API**：就业数据、行业统计
- **行业组织 API**：特定行业的公开数据
- **第三方数据聚合商**：如 Statista API、IBISWorld 数据

### 4. Web Scraping（合规方式）
- 对公开的行业报告、新闻进行合规抓取
- 确保遵守 robots.txt 和使用率限制

### 5. 社交媒体 API
- **Twitter API**：获取行业趋势和用户反馈
- **Reddit API**：获取社区讨论
- **LinkedIn API**：行业专业人士观点

## 技术架构

### 1. Agent系统
```
用户输入 → 工具调用 → 数据获取 → 综合分析 → 报告生成
```

#### 分析流程
1. **数据收集阶段**：调用各类数据工具获取市场数据
2. **竞争分析阶段**：分析竞争对手和市场格局
3. **趋势评估阶段**：分析行业趋势和未来方向
4. **报告生成阶段**：综合所有信息生成分析报告

### 2. 数据工具
- `search_market_size`：获取市场规模数据
- `search_competitors`：获取竞争对手信息
- `search_trends`：获取行业趋势
- `analyze_user_demand`：分析用户需求
- `fetch_financial_data`：获取财务数据
- `get_government_stats`：获取政府统计数据

### 3. 搜索服务
- 集成多个搜索引擎（Tavily、SerpAPI等）
- 搜索结果去重和结构化处理
- 搜索结果缓存机制
- 数据验证和时效性检查

### 4. 数据服务
- API 调用管理器
- 多源数据合并和去重
- 数据验证机制
- 错误处理和降级策略

### 5. 策略系统
- 市场进入策略
- 竞争分析策略
- 风险评估策略
- 机会识别策略

## 借鉴原项目的优势

1. **Agent架构**：使用大语言模型自主调用工具进行分析
2. **RAG机制**：通过搜索获取实时、准确的行业数据
3. **策略系统**：可扩展的策略框架，支持不同分析维度
4. **工具化设计**：模块化的工具系统，易于扩展和维护
5. **结构化输出**：标准化的报告格式，便于消费和展示
