# MarketScope - 行业分析AI应用

## 项目概述

MarketScope 是一个基于每日股票分析系统（daily_stock_analysis）项目架构开发的行业分析AI应用。该应用能够根据用户输入的APP类型和功能描述，分析相关行业现状，生成详细的市场分析报告。

## 项目目标

开发一个功能完整的行业分析AI应用，能够：
- 根据用户描述自动分析相关行业
- 获取最新的市场数据和竞争情况
- 生成专业的行业分析报告
- 提供市场进入的可行性评估和建议

## 核心功能

### 1. 输入处理
- 用户输入：APP类型 + 功能描述
- 示例：`"社交类APP，主要功能是兴趣小组聊天"`

### 2. 搜索增强（RAG）
- **市场规模分析**：搜索行业规模、用户数量、增长趋势
- **竞品分析**：识别主要竞争对手、功能对比、市场份额
- **趋势分析**：分析行业发展趋势、技术方向、政策影响
- **用户需求分析**：搜索用户痛点、需求变化、行为模式

### 3. 综合分析
- 市场容量评估
- 竞争格局分析
- 进入机会识别
- 风险因素评估

### 4. 报告生成
- 结构化的行业分析报告
- 市场进入可行性评估
- 建议策略和风险提示

## 技术架构

### 1. Agent系统
```
用户输入 → 工具调用 → 数据获取 → 综合分析 → 报告生成
```

#### 分析流程
1. **数据收集阶段**：调用各类搜索工具获取市场数据
2. **竞争分析阶段**：分析竞争对手和市场格局
3. **趋势评估阶段**：分析行业趋势和未来方向
4. **报告生成阶段**：综合所有信息生成分析报告

### 2. 数据工具
- `search_market_size`：获取市场规模数据
- `search_competitors`：获取竞争对手信息
- `search_trends`：获取行业趋势
- `analyze_user_demand`：分析用户需求

### 3. 搜索服务
- 集成多个搜索引擎（Tavily、SerpAPI等）
- 搜索结果去重和结构化处理
- 搜索结果缓存机制
- 数据验证和时效性检查

### 4. 策略系统
- 市场进入策略
- 竞争分析策略
- 风险评估策略
- 机会识别策略

## 借鉴原项目的优势

1. **Agent架构**：使用大语言模型自主调用工具进行分析
2. **RAG机制**：通过搜索获取实时、准确的行业数据
3. **策略系统**：可扩展的策略框架，支持不同分析维度
4. **工具化设计**：模块化的工具系统，易于扩展和维护
5. **结构化输出**：标准化的报告格式，便于消费和展示

## Agent工作流程

1. **接收用户输入**：获取用户提供的APP类型和功能描述
2. **智能工具选择**：根据输入内容选择合适的分析工具
3. **数据收集**：通过搜索服务获取相关行业数据
4. **多维度分析**：执行市场规模、竞争、趋势等分析
5. **报告生成**：将分析结果整合成结构化报告
6. **输出结果**：向用户提供完整的行业分析报告

## 设计原则

- **模块化**：各功能模块独立，便于维护和扩展
- **可扩展**：支持添加新的分析维度和工具
- **准确性**：通过RAG机制确保数据的实时性和准确性
- **用户友好**：提供清晰的报告和可操作的建议
